# Feedback Categorization Guide

## BLOCKER

- **Must** fix before merge
- Prevents merge in CI
- Examples: Security vulnerabilities, critical bugs, circular dependencies

## CRITICAL

- **Should** fix before merge
- Requires team discussion if not fixed
- Examples: Missing tests for new features, architectural violations

## SUGGESTION

- Nice to have
- Can be deferred to follow-up PR
- Examples: Performance optimizations, refactoring opportunities

## QUESTION

- Needs clarification from author
- Blocks merge until answered
- Examples: Unclear intent, potential bug, missing context

## PRAISE

- Highlight good patterns
- Improves team morale and learning
- Examples: Elegant solution, thorough tests, clear documentation
