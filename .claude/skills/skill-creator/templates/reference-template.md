## Guidelines for Using This Template

### When to Create a REFERENCE.md

Create a REFERENCE.md when:
- SKILL.md exceeds 500 lines
- You have detailed API documentation
- There are multiple implementation approaches to compare
- Platform-specific setup requires extensive explanation
- Advanced patterns need deep technical explanation

### What Goes in REFERENCE.md vs SKILL.md

**SKILL.md (procedural, high-level):**
- Quick start procedures
- Common workflows
- Essential configuration
- Basic examples

**REFERENCE.md (comprehensive, detailed):**
- Complete API documentation
- All configuration options
- Platform-specific setup guides
- Advanced patterns and edge cases
- Performance optimization
- Security considerations
- Troubleshooting deep dives

### Structure

Organize by topic, not by difficulty:
- Group related information together
- Use clear section headings
- Include a table of contents
- Link from SKILL.md to specific sections

### Code Examples

All code examples should:
- Be complete and runnable
- Include comments explaining non-obvious parts
- Show both simple and advanced usage
- Include error handling
- Demonstrate best practices
